Platform overview

What does RTrace do?

A quick orientation before the questions begin.

📊 DataTrace

Audits data files for encoding, headers, missing values, duplicate rows, FAIR compliance (Findable, Accessible, Interoperable, Reusable), and data provenance.

📝 DocsTrace

Checks README quality, vignettes, changelogs, citation files, contributing guides, and inline function documentation coverage.

📦 PackageQA

Validates DESCRIPTION completeness, license files, version format, NAMESPACE hygiene, test coverage presence, and NEWS.md format.

🔁 Reproducibility

Flags hardcoded paths, missing set.seed(), external downloads without caching, absent renv.lock, and un-cleaned temp files.

How it works: You run rtrace scan . in any R project and get a scored report — overall and per module — with exact file and line references for every issue. Reports export as console, HTML, Markdown, JSON, CSV, XML, or SARIF (for GitHub Code Scanning).
